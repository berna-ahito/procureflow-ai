# Plan 01-07: Requests Router, Approvals Router, Audit Router

## Objective

Build the three business-logic routers. Enforce IDOR protection on every request ID access. Route POST /requests/{id}/submit through the approval engine. Record all status changes in audit log. Register all routers in main.py.

<task id="T01" type="execute">
  <title>Create app/routers/requests.py — purchase request CRUD with IDOR</title>
  <read_first>
    - app/schemas/purchase_request.py — PurchaseRequestCreate, PurchaseRequestResponse, PurchaseRequestSummary
    - app/core/deps.py — get_current_active_user, require_role
    - app/db/models.py — PurchaseRequest, User
    - app/services/approval_engine.py — route_request
    - app/services/audit_service.py — log_action, ACTION_SUBMITTED, ACTION_ROUTED
    - .planning/REQUIREMENTS.md — FR-REQ-2 through FR-REQ-8, NFR-SEC-2
  </read_first>
  <action>
    Create app/routers/requests.py. Router prefix: /requests, tag: "requests"

    Helper function _get_request_or_403(db, request_id, current_user) -> PurchaseRequest:
      Query PurchaseRequest by id → 404 if not found.
      IDOR check:
        if current_user.role == "requester" and req.requester_id != current_user.id → raise HTTPException(403, "Access denied")
        if current_user.role == "manager" and req.assigned_role != "manager" and req.requester_id != current_user.id → 403
        if current_user.role == "finance" and req.assigned_role != "finance" and req.requester_id != current_user.id → 403
        if current_user.role == "admin" → always allow
      Return request.
      (Do NOT use 404 for unauthorized access — 403 prevents ID enumeration)

    POST /requests/ (requester role):
      current_user = Depends(require_role("requester"))
      Body: PurchaseRequestCreate
      Create PurchaseRequest with status="draft", requester_id=current_user.id
      db.add(); db.commit(); db.refresh()
      Return PurchaseRequestResponse

    GET /requests/ (authenticated users):
      current_user = Depends(get_current_active_user)
      Filter by role:
        requester → filter by requester_id == current_user.id
        manager → filter by assigned_role == "manager" OR requester_id == current_user.id
        finance → filter by assigned_role == "finance" OR requester_id == current_user.id
        admin → no filter (all requests)
      Return list[PurchaseRequestSummary]

    GET /requests/{request_id}:
      current_user = Depends(get_current_active_user)
      Call _get_request_or_403(db, request_id, current_user)
      Return PurchaseRequestResponse

    PATCH /requests/{request_id} (requester updating own draft):
      current_user = Depends(require_role("requester"))
      Call _get_request_or_403 (enforces ownership)
      If request.status != "draft" → raise HTTPException(400, "Can only edit draft requests")
      Apply non-None fields from PurchaseRequestUpdate body
      db.commit(); db.refresh()
      Return PurchaseRequestResponse

    POST /requests/{request_id}/submit (requester submitting draft):
      current_user = Depends(require_role("requester"))
      Call _get_request_or_403
      If request.status != "draft" → HTTPException(400, "Request is not in draft status")
      old_status = request.status
      request.status = "pending_review"
      db.commit()
      log_action(db, request.id, current_user.id, ACTION_SUBMITTED, old_status, "pending_review")
      Call route_request(db, request) → updates status to pending_approval or needs_rule
      log_action(db, request.id, None, ACTION_ROUTED, "pending_review", request.status)
      Return PurchaseRequestResponse
  </action>
  <acceptance_criteria>
    - app/routers/requests.py exists
    - _get_request_or_403() returns 403 (not 404) for IDOR violations
    - POST /requests/ sets status="draft" and requester_id=current_user.id (from JWT, not body)
    - GET /requests/ filters by role correctly
    - POST /requests/{id}/submit calls route_request() AND log_action() for both transitions
    - PATCH /requests/{id} rejects non-draft requests with 400
    - py -c "from app.routers.requests import router" exits 0
  </acceptance_criteria>
</task>

  <action>
    Create app/routers/approvals.py. Router prefix: /approvals, tag: "approvals"

    POST /approvals/{request_id}/decide (manager, finance, admin):
      current_user = Depends(require_role("manager", "finance", "admin"))
      Body: ApprovalDecisionCreate (decision + optional note)
      Query PurchaseRequest by request_id → 404 if not found
      Check current_user.role matches request.assigned_role OR current_user.role == "admin"
        → 403 if mismatch (wrong approver)
      Check request.status == "pending_approval" → 400 if not
      old_status = request.status
      Call apply_decision(db, request, body.decision, body.note) → validates transition internally
        If ValueError raised → convert to HTTPException(400, str(e))
      Create ApprovalDecision record: request_id, actor_id=current_user.id, decision, note
      db.add(decision_record); db.commit()
      log_action(db, request.id, current_user.id, ACTION_DECISION, old_status, request.status, body.note)
      Return ApprovalDecisionResponse

    GET /approvals/rules (admin only):
      current_user = Depends(require_role("admin"))
      Return list of all ApprovalRule ordered by priority ASC
      Return list[ApprovalRuleResponse]

    POST /approvals/rules (admin only):
      current_user = Depends(require_role("admin"))
      Body: ApprovalRuleCreate
      Create ApprovalRule, db.add, db.commit
      Return ApprovalRuleResponse

    IMPORTANT: Body for /decide endpoint is ApprovalDecisionCreate — it has only `decision` and `note`.
    It does NOT accept AI recommended_action as the decision — human must explicitly choose.
  </action>

## Verification

```powershell
py -m uvicorn app.main:app --host 0.0.0.0 --port 8000 &

# Login as requester
$alice_token = (Invoke-RestMethod -Uri http://localhost:8000/auth/login -Method POST -ContentType "application/json" -Body '{"email":"alice@test.com","password":"alice123"}').access_token

# Create request
$req = Invoke-RestMethod -Uri http://localhost:8000/requests/ -Method POST -ContentType "application/json" -Headers @{Authorization="Bearer $alice_token"} -Body '{"title":"New Laptop","description":"Need laptop for dev work","category":"IT","urgency":"high","quantity":1,"estimated_cost":800,"justification":"Current laptop too slow for builds"}'
echo "Created request id: $($req.id) status: $($req.status)"

# Submit request
$submitted = Invoke-RestMethod -Uri "http://localhost:8000/requests/$($req.id)/submit" -Method POST -Headers @{Authorization="Bearer $alice_token"}
echo "After submit: $($submitted.status) assigned_role: $($submitted.assigned_role)"

# IDOR check: login as bob, try to access alice's request
$bob_token = (Invoke-RestMethod -Uri http://localhost:8000/auth/login -Method POST -ContentType "application/json" -Body '{"email":"bob@test.com","password":"bob123"}').access_token
try {
    Invoke-RestMethod -Uri "http://localhost:8000/requests/$($req.id)" -Method GET -Headers @{Authorization="Bearer $bob_token"}
    echo "ERROR: should have been 403"
} catch { echo "IDOR protection works: $($_.Exception.Response.StatusCode)" }

# Admin can see audit log
$admin_token = (Invoke-RestMethod -Uri http://localhost:8000/auth/login -Method POST -ContentType "application/json" -Body '{"email":"admin@test.com","password":"admin123"}').access_token
$audit = Invoke-RestMethod -Uri "http://localhost:8000/audit/requests/$($req.id)" -Headers @{Authorization="Bearer $admin_token"}
echo "Audit entries: $($audit.Count)"
```
