# Plan 01-03: Pydantic Schemas

## Objective

Create all Pydantic v2 schemas used for request validation and response serialization. Use separate Create/Update/Response/AdminUpdate shapes per domain entity. No mass assignment — status and role fields are never on user-facing Create schemas.

## Tasks

    - .planning/REQUIREMENTS.md — FR-AUTH-1, FR-AUTH-3 for exact fields
    - app/db/models.py — User model fields for reference
  </read_first>
  <action>
    Create app/schemas/auth.py with:

    LoginRequest:
      email: EmailStr
      password: str

    Token (response):
      access_token: str

    - LoginRequest has fields: email (EmailStr), password (str)
    - Token has fields: access_token (str), token_type (str)
    - TokenData has fields: user_id (int), role (str)
    - py -c "from app.schemas.auth import LoginRequest, Token, TokenData" exits 0

<task id="T02" type="execute">
  <title>Create app/schemas/user.py</title>
  <read_first>
    - .planning/REQUIREMENTS.md — FR-USERS-1, FR-USERS-2, FR-USERS-3
    - app/db/models.py — User model for field parity
  </read_first>
  <action>
    Create app/schemas/user.py with these classes:

    UserCreate (for admin creating users):
      email: EmailStr
      password: str (plain, will be hashed in service)
      full_name: str
      role: Literal["requester", "manager", "finance", "admin"]
      department_id: Optional[int] = None

    UserUpdate (for user updating own profile — no role, no is_active):
      full_name: Optional[str] = None
      department_id: Optional[int] = None

    UserAdminUpdate (admin updating user — can change role and active status):
      full_name: Optional[str] = None
      role: Optional[Literal["requester", "manager", "finance", "admin"]] = None
      department_id: Optional[int] = None
      is_active: Optional[bool] = None

    UserResponse (safe response — no hashed_password):
      id: int
      email: EmailStr
      full_name: str
      role: str
      department_id: Optional[int]
      is_active: bool
      created_at: datetime
      model_config = ConfigDict(from_attributes=True)

    CurrentUser (used in JWT dependency — minimal):
      id: int
      email: str
      role: str
      is_active: bool
      model_config = ConfigDict(from_attributes=True)

    Imports: from pydantic import BaseModel, EmailStr, ConfigDict; from typing import Optional, Literal; from datetime import datetime
  </action>
  <acceptance_criteria>
    - app/schemas/user.py exists
    - UserCreate has role field (Literal with 4 values)
    - UserCreate does NOT have is_active or hashed_password
    - UserResponse has model_config = ConfigDict(from_attributes=True)
    - UserResponse does NOT expose hashed_password
    - py -c "from app.schemas.user import UserCreate, UserResponse, CurrentUser" exits 0
  </acceptance_criteria>
</task>

<task id="T03" type="execute">
  <title>Create app/schemas/purchase_request.py</title>
  <read_first>
    - .planning/REQUIREMENTS.md — FR-REQ-1, FR-REQ-3, FR-REQ-8 for exact fields and anti-mass-assignment rule
    - app/db/models.py — PurchaseRequest model fields
  </read_first>
  <action>
    Create app/schemas/purchase_request.py with these classes:

    VALID_URGENCY = Literal["low", "medium", "high", "critical"]
    VALID_STATUS = Literal["draft", "pending_review", "pending_approval", "needs_rule", "approved", "rejected", "needs_more_info"]

    PurchaseRequestCreate (requester submitting — NO status, NO requester_id, NO assigned_role):
      title: str (min_length=3, max_length=500)
      description: str (min_length=10)
      category: str (min_length=2, max_length=100)
      urgency: VALID_URGENCY
      quantity: int (gt=0)
      estimated_cost: float (gt=0.0)
      vendor_id: Optional[int] = None
      justification: str (min_length=10)

    PurchaseRequestUpdate (requester editing own draft — only mutable fields):
      title: Optional[str] = None
      description: Optional[str] = None
      category: Optional[str] = None
      urgency: Optional[VALID_URGENCY] = None
      quantity: Optional[int] = None (gt=0 if provided)
      estimated_cost: Optional[float] = None (gt=0.0 if provided)
      vendor_id: Optional[int] = None
      justification: Optional[str] = None
      (NO status, NO assigned_role — prevents mass assignment)

    PurchaseRequestResponse:
      id: int
      title: str
      description: str
      category: str
      urgency: str
      quantity: int
      estimated_cost: float
      vendor_id: Optional[int]
      justification: str
      status: str
      requester_id: int
      assigned_role: Optional[str]
      created_at: datetime
      updated_at: datetime
      model_config = ConfigDict(from_attributes=True)

    PurchaseRequestSummary (list view — subset of fields):
      id: int
      title: str
      category: str
      urgency: str
      status: str
      estimated_cost: float
      requester_id: int
      created_at: datetime
      model_config = ConfigDict(from_attributes=True)

    Imports: from pydantic import BaseModel, ConfigDict, Field; from typing import Optional, Literal; from datetime import datetime
  </action>
  <acceptance_criteria>
    - app/schemas/purchase_request.py exists
    - PurchaseRequestCreate has NO status, requester_id, or assigned_role fields
    - PurchaseRequestCreate.quantity has gt=0 constraint
    - PurchaseRequestCreate.estimated_cost has gt=0.0 constraint
    - PurchaseRequestResponse includes: id, status, requester_id, assigned_role, created_at, updated_at
    - PurchaseRequestResponse has model_config = ConfigDict(from_attributes=True)
    - py -c "from app.schemas.purchase_request import PurchaseRequestCreate, PurchaseRequestResponse" exits 0
  </acceptance_criteria>
</task>
