# Plan 01-02: Database Models + Alembic Migration

## Objective

Define all SQLAlchemy ORM models for Phase 1 and create the initial Alembic migration that builds every table from scratch. No seed data yet. No AI review table yet (Phase 2).

## Tasks

<task id="T01" type="execute">
  <title>Create app/db/base.py — SQLAlchemy engine and session factory</title>
  <read_first>
    - app/core/config.py — for settings.database_url
    - requirements.txt — confirms sqlalchemy is available
  </read_first>
  <action>
    Create app/db/base.py with:
    - Declarative base: Base = declarative_base()
    - Engine: engine = create_engine(settings.database_url, connect_args={"check_same_thread": False})
      (check_same_thread=False required for SQLite with FastAPI)
    - SessionLocal factory: SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)
    - get_db() generator function that yields a session and closes it in finally block
    Imports needed: from sqlalchemy import create_engine; from sqlalchemy.orm import sessionmaker, declarative_base; from app.core.config import settings
    Export: Base, engine, SessionLocal, get_db
  </action>
  <acceptance_criteria>
    - app/db/base.py exists
    - Base, engine, SessionLocal, get_db all importable from app.db.base
    - engine uses settings.database_url (not hardcoded sqlite path)
    - connect_args={"check_same_thread": False} present on engine
    - get_db() is a generator (uses yield)
    - No circular imports when running: py -c "from app.db.base import Base, get_db"
  </acceptance_criteria>
</task>

<task id="T02" type="execute">
  <title>Create app/db/models.py — all ORM models</title>
  <read_first>
    - docs/DATA_MODEL.md — table list and field concepts
    - .planning/REQUIREMENTS.md — FR-USERS-1, FR-REQ-1, FR-APPR-1, FR-AUDIT-2 for exact field names
    - app/db/base.py — for Base import
  </read_first>
  <action>
    Create app/db/models.py. Define these SQLAlchemy models in order (to avoid FK ordering issues):

    1. Department:
       id: Integer PK, name: String(100) unique not null, created_at: DateTime default utcnow

    2. User:
       id: Integer PK, email: String(255) unique not null, hashed_password: String(255) not null,
       full_name: String(255) not null, role: String(50) not null (values: requester|manager|finance|admin),
       department_id: Integer FK→departments.id nullable, is_active: Boolean default True,
       created_at: DateTime default utcnow

    3. Vendor:
       id: Integer PK, name: String(255) not null, contact_email: String(255) nullable,
       is_active: Boolean default True, created_at: DateTime default utcnow

    4. PurchaseRequest:
       id: Integer PK, title: String(500) not null, description: Text not null,
       category: String(100) not null, urgency: String(50) not null (low|medium|high|critical),
       quantity: Integer not null, estimated_cost: Float not null,
       vendor_id: Integer FK→vendors.id nullable, justification: Text not null,
       status: String(50) not null default "draft",
       requester_id: Integer FK→users.id not null,
       assigned_role: String(50) nullable,
       created_at: DateTime default utcnow, updated_at: DateTime default utcnow onupdate utcnow

    5. ApprovalRule:
       id: Integer PK, name: String(255) not null,
       min_amount: Float not null, max_amount: Float nullable (null = no upper bound),
       category: String(100) nullable (null = matches any category),
       required_role: String(50) not null (manager|finance|admin),
       priority: Integer not null (lower = higher priority),
       is_active: Boolean default True, created_at: DateTime default utcnow

    6. ApprovalDecision:
       id: Integer PK, request_id: Integer FK→purchase_requests.id not null,
       actor_id: Integer FK→users.id not null,
       decision: String(50) not null (approved|rejected|needs_more_info),
       note: Text nullable, created_at: DateTime default utcnow

    7. AuditLog:
       id: Integer PK, request_id: Integer FK→purchase_requests.id not null,
       actor_id: Integer FK→users.id nullable (null = system action),
       action: String(100) not null,
       old_status: String(50) nullable, new_status: String(50) nullable,
       note: Text nullable, created_at: DateTime default utcnow

    Table names (via __tablename__): departments, users, vendors, purchase_requests, approval_rules, approval_decisions, audit_logs.
    Import Base from app.db.base. Use sqlalchemy Column, Integer, String, Float, Boolean, Text, DateTime, ForeignKey.
    from datetime import datetime, timezone — use datetime.now(timezone.utc) for defaults.
    Do NOT define ai_reviews table here (Phase 2).
  </action>
  <acceptance_criteria>
    - app/db/models.py exists
    - Defines 7 classes: Department, User, Vendor, PurchaseRequest, ApprovalRule, ApprovalDecision, AuditLog
    - All FK references use string table.column format (e.g., "users.id")
    - PurchaseRequest.status has default="draft"
    - AuditLog has no UPDATE or DELETE hooks (append-only intent clear from model)
    - py -c "from app.db.models import User, PurchaseRequest, ApprovalRule, AuditLog" exits 0
    - No model is over 30 fields
  </acceptance_criteria>
</task>

<task id="T03" type="execute">
  <title>Initialize Alembic and configure env.py</title>
  <read_first>
    - app/db/base.py — for Base and engine imports
    - app/db/models.py — must be imported in env.py so models register with Base.metadata
    - .planning/ROADMAP.md — confirms alembic/ directory structure
  </read_first>
  <action>
    Run: py -m alembic init alembic
    This creates: alembic.ini, alembic/env.py, alembic/script.py.mako, alembic/versions/

    Then edit alembic/env.py:
    - Import Base from app.db.base: from app.db.base import Base
    - Import all models so they register: from app.db import models as _models  # noqa: F401
    - Set target_metadata = Base.metadata
    - For SQLite offline mode, set sqlalchemy.url in alembic.ini to use env var:
      In env.py run_migrations_offline(), use: config.get_main_option("sqlalchemy.url")
    - In env.py run_migrations_online(), use engine from app.db.base:
      from app.db.base import engine; with engine.connect() as connection: ...

    Edit alembic.ini:
    - Set: sqlalchemy.url = sqlite:///./procureflow.db
      (This is the default; env var override via env.py is the production path)
  </action>
  <acceptance_criteria>
    - alembic.ini exists at project root
    - alembic/env.py imports Base from app.db.base
    - alembic/env.py imports models module (so all models are registered)
    - alembic/env.py sets target_metadata = Base.metadata
    - alembic/versions/ directory exists
    - py -c "from alembic import config" exits 0
  </acceptance_criteria>
</task>
