# Plan 01-05: Auth Router, Users Router, and Seed Data

## Objective

Build the auth and users routers, register them in main.py, and create a seed script that populates demo users and departments for development/testing. Seed is idempotent (re-running doesn't duplicate records).

## Tasks

<task id="T01" type="execute">
  <title>Create app/routers/auth.py — login endpoint</title>
  <read_first>
    - app/core/security.py — verify_password, create_access_token
    - app/schemas/auth.py — LoginRequest, Token schemas
    - app/db/base.py — get_db
    - app/db/models.py — User model
    - .planning/REQUIREMENTS.md — FR-AUTH-1 exact contract
  </read_first>
  <action>
    Create app/routers/auth.py:

    Router prefix: /auth, tag: "auth"

    POST /auth/login:
      Request body: LoginRequest (email, password)
      Logic:
        1. Query db for User where email == request.email and is_active == True
        2. If not found or verify_password fails → raise HTTPException(401, "Invalid credentials")
        3. Call create_access_token({"user_id": user.id, "role": user.role})
        4. Return Token(access_token=token, token_type="bearer")
      Response model: Token
      No Depends(get_current_user) on this endpoint (it's the login endpoint)

    Use APIRouter(prefix="/auth", tags=["auth"])
    DB session via: db: Session = Depends(get_db)
  </action>
  <acceptance_criteria>
    - app/routers/auth.py exists
    - POST /auth/login endpoint defined
    - Returns Token schema on success (access_token + token_type)
    - Raises HTTPException(401) for wrong password
    - Raises HTTPException(401) for unknown email
    - Does NOT accept user_id in the request body
    - py -c "from app.routers.auth import router" exits 0
  </acceptance_criteria>
</task>

<task id="T02" type="execute">
  <title>Create app/routers/users.py — current user and admin user management</title>
  <read_first>
    - app/core/deps.py — get_current_active_user, require_role
    - app/schemas/user.py — UserResponse, UserCreate, UserAdminUpdate
    - app/db/models.py — User model
    - .planning/REQUIREMENTS.md — FR-USERS-3 (admin creates/deactivates users)
  </read_first>
  <action>
    Create app/routers/users.py:

    Router prefix: /users, tag: "users"

    GET /users/me:
      current_user: User = Depends(get_current_active_user)
      Return UserResponse.model_validate(current_user)
      No role restriction — any authenticated active user can see their own profile.

    POST /users/ (admin only):
      current_user: User = Depends(require_role("admin"))
      Body: UserCreate
      Hash the password using get_password_hash(body.password)
      Create User ORM object (map all fields, use hashed_password)
      Check email uniqueness: query db for existing user with same email → 400 if exists
      db.add(user); db.commit(); db.refresh(user)
      Return UserResponse.model_validate(user)

    PATCH /users/{user_id} (admin only — deactivate or change role):
      current_user: User = Depends(require_role("admin"))
      Body: UserAdminUpdate
      Query user by id → 404 if not found
      Apply only non-None fields from body (no mass assignment)
      db.commit(); db.refresh(user)
      Return UserResponse.model_validate(user)

    Do NOT expose password hash in any response.
  </action>
  <acceptance_criteria>
    - app/routers/users.py exists
    - GET /users/me uses Depends(get_current_active_user) not request body
    - POST /users/ uses Depends(require_role("admin"))
    - POST /users/ hashes password before storing
    - POST /users/ returns 400 on duplicate email
    - PATCH /users/{user_id} applies only provided fields (partial update)
    - No endpoint returns hashed_password field
    - py -c "from app.routers.users import router" exits 0
  </acceptance_criteria>
</task>

## Verification

```powershell
# Start server
py -m alembic upgrade head
py scripts/seed.py
py -m uvicorn app.main:app --host 0.0.0.0 --port 8000 &

# Test login
curl -s -X POST http://localhost:8000/auth/login -H "Content-Type: application/json" -d "{\"email\":\"alice@test.com\",\"password\":\"alice123\"}"
# Expected: {"access_token":"...","token_type":"bearer"}

# Test wrong password
curl -s -X POST http://localhost:8000/auth/login -H "Content-Type: application/json" -d "{\"email\":\"alice@test.com\",\"password\":\"wrong\"}"
# Expected: 401

# Test /me with token
TOKEN=$(curl -s -X POST http://localhost:8000/auth/login -H "Content-Type: application/json" -d "{\"email\":\"alice@test.com\",\"password\":\"alice123\"}" | py -c "import sys,json; print(json.load(sys.stdin)['access_token'])")
curl -s http://localhost:8000/users/me -H "Authorization: Bearer $TOKEN"
# Expected: {"id":1,"email":"alice@test.com","role":"requester",...}
```
